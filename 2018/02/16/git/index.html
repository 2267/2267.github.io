git使用指南

Date: February 16, 2018  18:48:05 Category: 工具使用

配置

所有用户的配置文件：/etc/gitconfig
当前用户的全局配置文件：~/.gitconfig 或 ~/.config/git/config
当前仓库的配置文件：<git repo>/.git/config

修改.gitignore文件，可以指定那些文件不会被git跟踪，如果已经跟踪了，就用git rm –cached

git只能用来跟踪和比较文本文件

使用

  • 下载可以访问的库源代码，git clone <url>，url有两种形式https和ssh，使用ssh每次push不用再次输入密码
    • https: https://github.com/0xNi/xxx.git
    • ssh:
      • git clone ssh://<user>@<ip>:port/<absolute path>
      • git clone ssh://<user>@<ip>:port/~/<home relative path>
      • git clone <user>@<ip>:<home relative path>
  • 创建本地git仓库：
    • git init
    • git add <file> 将改动添加到暂存区
    • git commit -m <string> 将暂存区更新到仓库中
  • 有用的命令：
    • git status 查看改动状态
    • git diff 列出已经修改却没有加入到暂存区的内容，工作区vs暂存区
    • git diff –cached 列出加入到暂存区的已经修改内容，暂存区vs仓库
    • git diff HEAD – <file> 比较工作区和仓库的差异
    • git diff <ver> – <file> 比较当前工作区和某一版本的仓库的差异
    • git diff <ver1> <ver2> 查看任意两个版本之间的改动：
    • git diff <ver1> <ver2> <file> 比较两个版本号码的文件(夹)的差异
    • git log 查看所有的commit，包含commit id
    • git log –pretty=oneline 一行一个log
    • git reflog 有时git reset后看不到后来改动的commit id，使用这个命令可以看到。记录命令历史
  • 其他控制命令：
    • git reset –hard HEAD^ 回到上一次commit的状态,
      • HEAD：当前的版本，HEAD或HEAD^：上一版本，HEAD2或HEAD^^上两个的版本
    • git reset HEAD 撤销掉加入暂存区的修改
    • git reset –hard abf884471f932666a171cf933a9785cc48a45d80 回到commit id指定的状态
    • git reset HEAD <file> 撤销暂存区某个文件的修改
    • git rm <file> 删除暂存区里面的内容以及该文件
    • git rm –cached <file> 仅删除暂存区里面的内容，commit后文件会变成untracked状态
    • git checkout – <file> 撤销文件修改，文件变成上一次commit后的状态
    • git tag <tag name> 给当前版本的仓库打一个标签
    • git tag <tag name> <commit id> 给某一版本打标签
    • git tag -a <tag name> -m <msg> 打标签的同时添加一些信息
    • git tag 以字母顺序而非时间顺序列出标签
    • git show <tag name> 列出标签的信息，包括commit和diff，以及-a添加的信息
    • git tag -d <tag name> 删除标签

远程控制

  1. 创建ssh key ssh-keygen -t rsa -C [email]
  2. 将ssh key id_isa.pub添加到远程服务器上.ssh/authorized_keys
  3. 远程：
  4. 本地：（一般来说本地的默认分支为master，远程的为origin）
  • 其他命令
    • git remote 查看远程库的信息
    • git remote -v 查看更详细的信息
    • git push origin dev 推送其他分支，比如dev
    • git push origin <tag name> 提交tag到远程
    • git push origin –tags 提交所有tag到远程
    • git push origin :refs/tags/<tag name> 远端删除某一个tag
    • 远程库不为空时，先用git pull把最新的提交从origin/dev抓下来，然后，在本地合并，解决冲突，再推送，
    • git pull = git fetch + git merge
    • git pull –rebase = git fetch + git rebase
