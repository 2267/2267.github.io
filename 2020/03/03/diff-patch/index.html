diff, patch 文件差异比较及打补丁

Date: March 3, 2020  10:04:46 Category: 工具使用

假设存在两个文件夹，old和new，new是在old的代码上进行了一系列的修改的新文件夹

diff -ruN old new > 1.patch， 这里的 patch文件包含了不同的内容的信息

假设有人同样有old文件夹，将patch分享给他，它可以将old文件夹和patch文件放在同一文件夹后
patch -s -p0 < 1.patch

或者进入old文件夹，把patch文件放在old中之后
patch -s -p1 < 1.patch

解释：
diff

  • r：递归的比较，用于文件夹
  • u：貌似不加这个参数也行，据说的增加可读性
  • N：不带这个参数的话，new中有新文件，新文件的内容不会被记录在patch中（diff的输出里）

patch

  • s：只显示错误的信息
  • p：去除diff输出中文件夹路径的层级数目
